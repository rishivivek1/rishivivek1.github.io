    Travel Map Data Update Prompt

    Attach this file along with your Google Takeout Visited.csv (or any CSV with the same format) and ask the AI to update the data files.

    Prompt

    You are given a CSV file of places I have visited (exported from Google Maps via Google Takeout). Parse it and regenerate the three JSON data files for my travel map page located in docs/mymap/.

    Input CSV Format

    The CSV has this structure:

    Places visited

Title,Note,URL,Tags,Comment
,,,,
Rock Beach,,https://www.google.com/maps/place/Rock+Beach/data=...,,
Kala Patthar Beach,Quite place to sit across the beach.,https://www.google.com/maps/place/...,,
    • Row 1: Header text "Places visited" (skip).
    • Row 2: Empty (skip).
    • Row 3: Column headers: Title,Note,URL,Tags,Comment.
    • Row 4: Empty data row (skip).
    • Row 5+: Actual place entries.
    • The Title column contains the place name.
    • The Note column contains an optional personal note (may be empty, may span multiple lines enclosed in quotes).
    • The URL, Tags, and Comment columns are not used.
    • Some titles are quoted and may contain commas.
    • Some notes contain newlines (the CSV uses standard quoting for multi-line fields).

    Step 1: Extract Places

    From each row, extract: - name: The Title value. Clean it up: - Remove suffixes like "Recently viewed". - Trim whitespace. - Keep proper capitalization as-is. - note: The Note value. Clean it up: - Replace literal newlines with spaces or ". " to make a single line. - Trim whitespace. - Empty string if no note.

    Step 2: Map Each Place to a City

    For every place, determine which city/town it belongs to using your geographic knowledge:

    • A city is a town, city, or distinct destination (e.g., Jaisalmer, Port Blair, Havelock Island, Udupi).
    • Landmarks, beaches, temples, forts, parks etc. within the same city are grouped under that city.
    • Examples of groupings:
    • Rock Beach → Puducherry
    • Sam Sand Dunes, Kuldhara, Longewala War Memorial, Tanot Mata Temple → Jaisalmer
    • Cellular Jail, Corbyn's Cove Beach, Chidiya Tapu Beach → Port Blair
    • Kapu Beach, Malpe Beach, Kapu Light House, Bramavara → Udupi
    • Kala Patthar Beach, Radhanagar Beach, Vijay Nagar Beach → Havelock Island
    • Nandi Hills, Skandagiri Hills → Nandi Hills (not Bengaluru — these are distinct hill destinations)
    • Rohtang La → Manali
    • Isha Yoga Center → Coimbatore
    • If a place IS a city (e.g., "Bengaluru", "Kolkata"), that city entry has a single place with the same name.
    • If unsure, use the nearest well-known town.

    Step 3: Map Each City to a State/UT and Country

    • Determine the Indian state or union territory for each city.
    • Determine the country for each city.
    • For places outside India, identify the correct country (e.g., Bali → Indonesia, Kuta → Indonesia).

    Step 4: Generate cities.json

    Output format:

    [
  {
    "city": "CityName",
    "places": [
      {"name": "Place 1", "note": "Optional note"},
      {"name": "Place 2", "note": ""}
    ]
  }
]

    Rules: - Sort cities alphabetically by city name. - Within each city, list places in the order they appear in the CSV (preserves chronological order). - Each place must appear in exactly one city. - Every place from the CSV must be included (no drops). - Notes must be single-line strings (replace newlines). - Use empty string "" for missing notes, never null. - Use consistent formatting: 2-space indentation, one city object per block.

    Step 5: Generate states.json

    Output format:

    [
  "Andaman and Nicobar Islands",
  "Assam",
  "Bihar"
]

    Rules: - Flat array of unique state/UT names. - Sorted alphabetically. - Include union territories (e.g., "Puducherry", "Andaman and Nicobar Islands"). - Only Indian states/UTs — do not include states from other countries.

    Step 6: Generate countries.json

    Output format:

    [
  "India",
  "Indonesia"
]

    Rules: - Flat array of unique country names. - Sorted alphabetically.

    Validation Checklist

    After generating, verify: - [ ] Total places across all cities in cities.json equals total data rows in the CSV. - [ ] No duplicate places (each CSV row appears exactly once). - [ ] Every city in cities.json maps to at least one state in states.json. - [ ] Every city in cities.json maps to at least one country in countries.json. - [ ] All arrays are sorted alphabetically where specified. - [ ] No null values — use "" for empty notes. - [ ] JSON is valid and properly formatted.

    Output

    Write the three files: - docs/mymap/cities.json - docs/mymap/states.json - docs/mymap/countries.json

    Print a summary table:

    Metric Count
    Total places ?
    Unique cities ?
    States/UTs ?
    Countries ?

    And list any places where the city assignment was uncertain.
